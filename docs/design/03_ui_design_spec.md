## 1. デザイン原則

### 1.1 コアバリュー

| 原則 | 説明 | 実装指針 |
|------|------|---------|
| **シンプルさ第一** | 美容室スタッフがITに詳しくなくても使える | 1画面1アクション。設定より規約。迷ったら選択肢を減らす |
| **コピペ最適化** | HPBへの転記がゴール | コピーボタンを最も目立つ位置に。コピー後のフィードバック明確に |
| **スタイリストの個性** | 全員が同じ文章にならない | メタデータの可視化。生成前に「この人らしさ」が確認できる |
| **信頼性** | AIの限界を正直に伝える | HPB規約警告を適切に表示。「確認してから掲載」を促す |

### 1.2 デザインシステム

- **UIライブラリ**: shadcn/ui（Radix UI + Tailwind CSS）
- **カラーパレット**: 美容業界を意識した洗練されたトーン（後述）
- **タイポグラフィ**: Noto Sans JP（日本語）+ Inter（英数字）
- **アイコン**: Lucide Icons
- **スペーシング**: 4px グリッドシステム

### 1.3 カラーパレット（案）

```
Primary:      #8B5CF6 (Violet 500)     — メインアクション、ブランドカラー
Primary Dark: #7C3AED (Violet 600)     — ホバー状態
Secondary:    #F472B6 (Pink 400)       — アクセント、Pro機能
Surface:      #FAFAFA                  — 背景
Card:         #FFFFFF                  — カード背景
Border:       #E5E7EB (Gray 200)      — ボーダー
Text:         #1F2937 (Gray 800)      — 本文

```

## 3. コンポーネント設計

### 3.1 共通コンポーネント

| コンポーネント | 用途 | 備考 |
|-------------|------|------|
| `CopyButton` | ワンクリックコピー | 最重要UI。成功フィードバック付き |
| `CharCounter` | 文字数カウンター | HPB上限表示、超過時は赤色警告 |
| `HpbPreview` | HPB風プレビュー | コンテンツ種別ごとにレイアウト切替 |
| `StylistSelector` | スタイリスト選択 | アバター+名前+得意メニュー表示 |
| `ContentTypeSelector` | コンテンツ種別選択 | 文字数上限とPro制限表示 |
| `ChatUI` | チャット修正 | ストリーミング表示、履歴スクロール |
| `PlanBadge` | Pro/Free表示 | Freeは制限表示、Proは解放表示 |
| `UsageMeter` | 利用量メーター | 月間残り回数の可視化 |
| `HpbWarning` | HPB規約警告 | AI画像のHPB掲載不可警告 |

### 3.2 CopyButtonの実装仕様

```
[状態遷移]
Default: 「📋 テキストをコピー」（Primary色ボタン）
  ↓ クリック
Copying: クリップボードAPIでコピー実行
  ↓ 成功
Success: 「✅ コピーしました！」（Success色、2秒間）
  ↓ 2秒後
Default に戻る

[失敗時]
Error: 「❌ コピーに失敗しました」（Danger色、3秒間）
  → フォールバック: テキスト全選択状態にする
```

### 3.3 CharCounterの実装仕様

```
[表示]
"156 / 200文字"

[状態]
- 80%未満: グレーテキスト + 緑プログレスバー
- 80〜99%: オレンジテキスト + オレンジプログレスバー + 「あと44文字」
- 100%: 赤テキスト + 赤プログレスバー + 「⚠️ 文字数超過！」
- 超過時: 自動短縮ボタン表示「✂️ 自動で収める」
```

## 4. レスポンシブ対応

| ブレークポイント | 対応 |
|----------------|------|
| Desktop (1024px+) | フル機能。サイドバー + メインコンテンツ |
| Tablet (768-1023px) | サイドバーは折りたたみ。チャット修正は全画面 |
| Mobile (〜767px) | 閲覧・コピーのみ対応。生成操作はPC推奨の案内 |

## 5. アクセシビリティ

- キーボードナビゲーション対応（Tab順序の管理）
- コピーボタンに`aria-label`付与
- 色だけに依存しない状態表現（アイコン + テキスト + 色）
- フォーカスリングの明示的表示
- スクリーンリーダー対応（shadcn/uiのRadix UIが対応済み）

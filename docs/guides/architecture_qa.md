## 1. システム全体像

### Q: HPB Content Studioのシステム構成を一言で教えてください

**A:** フロントエンド（React）とバックエンド（Python FastAPI）を分離したモダンなWebアプリです。バックエンドからClaude AI APIを呼び出してテキストを生成し、データはSupabase（PostgreSQL）に保存しています。

```
ユーザー（ブラウザ）
    │
    ▼
┌─────────────────────┐     ┌──────────────────────┐     ┌──────────────┐
│  フロントエンド       │     │  バックエンド API      │     │  外部API      │
│  React + TypeScript  │────▶│  Python FastAPI       │────▶│  Claude AI   │
│  Vercel でホスティング│     │  Railway でホスティング │     │  (Anthropic) │
└─────────────────────┘     └──────────┬───────────┘     └──────────────┘
                                       │
                                       ▼
                            ┌──────────────────────┐
                            │  データベース + 認証    │
                            │  Supabase             │
                            │  (PostgreSQL + Auth)   │
                            └──────────────────────┘
```

### Q: 各サービスの役割は？

| サービス | 役割 | URL |
|---------|------|-----|
| **Vercel** | フロントエンド配信（React SPA） | `hpb-content-studio.vercel.app` |
| **Railway** | バックエンドAPI実行（FastAPI） | `hpb-content-studio-production.up.railway.app` |
| **Supabase** | データベース（PostgreSQL）+ ユーザー認証 | `xxxxx.supabase.co` |
| **Anthropic** | AI テキスト生成 API（Claude 4.5 Haiku） | `api.anthropic.com` |

## 2. インフラ・ホスティング

### Q: なぜこの構成を選んだのですか？

**A:** 個人開発 × MVP フェーズに最適化した構成です。

| 選定基準 | 判断 |
|---------|------|
| **コスト** | 月額 ¥2,000〜5,000 で本番運用可能 |
| **運用負荷** | サーバー管理不要（フルマネージド） |
| **スケーラビリティ** | ユーザー増加時もサービス側で自動スケール |
| **デプロイ速度** | git push → 自動デプロイ（数分で反映） |

### Q: Railway とは何ですか？なぜ選びましたか？

**A:** Railwayはクラウドのアプリケーション実行プラットフォームです。AWSやGCPより圧倒的に簡単で、Herokuの後継として人気があります。

**選定理由:**
- **常時稼働** — 無料プランでもスリープしない（Renderは15分無通信でスリープ）
- **自動デプロイ** — GitHubにpushするだけで本番に反映
- **Dockerfile対応** — 環境の再現性が高い
- **従量課金** — 使った分だけ。低トラフィック時はコストが低い

### Q: サーバーは常時稼働していますか？

**A:** はい。Railway上のバックエンドAPIは24時間365日稼働しています。ユーザーがいつアクセスしてもレスポンスが返ります。スリープやコールドスタートはありません。

### Q: 月額コストはいくらですか？

| サービス | 月額目安 | 備考 |
|---------|---------|------|
| **Railway**（バックエンド） | $5〜10（約¥750〜1,500） | Hobbyプラン $5基本料 + 従量 |
| **Vercel**（フロントエンド） | $0 | 無料プランで十分 |
| **Supabase**（DB + 認証） | $0 | 無料プランで十分（500MB DB, 50K MAU） |
| **Anthropic**（Claude AI） | ¥100〜1,000 | 利用量に応じた従量課金 |
| **合計** | **約¥1,000〜3,000/月** | MVP〜初期ユーザーフェーズ |

## 3. セキュリティ

### Q: ユーザーのデータはどう保護されていますか？

**A:** 多層防御で保護しています。

| レイヤー | 対策 |
|---------|------|
| **通信** | 全経路HTTPS（TLS暗号化） |
| **認証** | Supabase Auth（JWT トークンベース） |
| **データ分離** | Row Level Security（RLS）で他ユーザーのデータにアクセス不可 |
| **APIキー** | 環境変数で管理、コードに埋め込まない |
| **CORS** | 許可されたフロントエンドURLからのみアクセス可能 |

### Q: Row Level Security（RLS）とは？

**A:** PostgreSQLのデータベースレベルのアクセス制御です。SQLのSELECT/INSERT/UPDATE/DELETEに対して「自分のデータだけ」というルールをDB自体に強制します。

たとえアプリケーションにバグがあっても、**DB側でデータ漏洩を防止**できます。これはSupabaseの推奨設計パターンで、全テーブルに適用済みです。

### Q: AI（Claude）に送るデータはどうなりますか？

**A:** Anthropicの利用規約上、APIから送信されたデータは**モデルの学習に使用されません**。送信されるのはサロン情報・スタイリスト情報・ユーザーの追加指示のみで、個人の顧客情報は送信されません。

## 4. AI テキスト生成

### Q: どのAIモデルを使っていますか？

**A:** テキスト生成には **Claude 4.5 Haiku**（Anthropic社）を使用しています。

| 項目 | 詳細 |
|------|------|
| モデル | Claude 4.5 Haiku（`claude-haiku-4-5-20251001`） |
| 選定理由 | 日本語品質が高い、高速レスポンス、低コスト |
| 出力方式 | SSE（Server-Sent Events）ストリーミング |
| 1回あたりのAPI費用 | 約¥0.3〜2（生成するテキストの長さによる） |

### Q: テキスト生成の仕組みは？

```
① ユーザーがコンテンツタイプ（ブログ/紹介文等）と追加指示を入力
    ↓
② バックエンドがプロンプトを自動合成
   [システムプロンプト] + [サロン情報] + [スタイリスト情報] + [HPB文字数制約] + [追加指示]
    ↓
③ Claude APIにストリーミングリクエスト
    ↓
④ 生成テキストをリアルタイムでフロントエンドに返却（SSE）
    ↓
⑤ 生成結果 + 使用プロンプトをDBに保存
```

### Q: なぜストリーミング出力なのですか？

**A:** ユーザー体験のためです。AI生成には5〜15秒かかりますが、ストリーミングなら1〜2秒で最初の文字が表示され始めます。「ちゃんと動いてる」という安心感と、完成を待つ時間のストレス軽減になります。

## 5. 可用性・障害

### Q: サービスが落ちるのはどんな時ですか？

| 障害パターン | 影響 | 発生可能性 | 対処 |
|------------|------|-----------|------|
| **Railway障害** | API全体が停止 | 低（稼働率99.9%+） | Railway Status で確認、復旧待ち |
| **Supabase障害** | DB/認証が停止 | 低（稼働率99.9%+） | Supabase Status で確認、復旧待ち |
| **Vercel障害** | フロントエンドが表示されない | 極めて低 | Vercel Status で確認 |
| **Anthropic API障害** | テキスト生成のみ不可 | 稀に発生 | エラーメッセージ表示、リトライ |
| **デプロイ時の不具合** | 新バージョンでクラッシュ | 開発中は起きうる | Railway で前バージョンにロールバック |
| **APIキー期限切れ/超過** | 該当機能が停止 | 管理ミスで発生 | キーの更新、クレジット補充 |

### Q: 全部が同時に落ちることはありますか？

**A:** 極めて低い確率です。Vercel（CDN）、Railway（アプリ）、Supabase（DB）、Anthropic（AI）はすべて独立した事業者のサービスです。全社が同時に障害を起こす可能性はほぼゼロです。

### Q: データのバックアップはありますか？

**A:** Supabaseが自動的にデータベースの日次バックアップを取得しています（Proプラン以上でPoint-in-Time Recovery対応）。現在のFreeプランでも日次バックアップは有効です。

### Q: 障害時のユーザーへの影響は？

| コンポーネント停止 | ユーザーへの影響 |
|------------------|----------------|
| Railway（API）停止 | ログイン・生成・保存すべて不可。フロントエンドの画面自体は表示される |
| Supabase停止 | ログイン不可、データ読み書き不可 |
| Anthropic API停止 | テキスト生成のみ不可。それ以外の機能（履歴閲覧、サロン編集等）は正常 |
| Vercel停止 | 画面が表示されない。APIは生きている |

## 6. スケーラビリティ

### Q: ユーザーが増えても大丈夫ですか？

**A:** はい。現在の構成で数百〜数千ユーザーまで対応可能です。

| フェーズ | ユーザー数 | 必要な対応 |
|---------|-----------|-----------|
| MVP（現在） | 〜100人 | 対応不要。現状の構成で十分 |
| 成長期 | 100〜1,000人 | Supabase Proプラン（$25/月）に切り替え |
| 拡大期 | 1,000〜10,000人 | Railway のインスタンスサイズ拡大、API レート制限の調整 |
| 本格展開 | 10,000人〜 | 専用インフラへの移行を検討（AWS/GCP） |

### Q: 同時アクセスが集中したらどうなりますか？

**A:** テキスト生成はAI APIの呼び出しを伴うため、同時100リクエスト程度が現実的な上限です。ただし、利用パターン（美容室スタッフが業務時間中に使う）を考慮すると、同時アクセスの集中は起きにくい構造です。

## 7. デプロイ・運用

### Q: コードの更新はどうやって本番に反映しますか？

```
開発者がコードを修正
    ↓
git push origin main
    ↓
GitHub が変更を検知
    ↓
┌──────────────┐    ┌──────────────┐
│ Vercel       │    │ Railway      │
│ 自動ビルド    │    │ 自動ビルド    │
│ フロントエンド│    │ バックエンド  │
│ 1〜2分で完了  │    │ 2〜3分で完了  │
└──────────────┘    └──────────────┘
    ↓                    ↓
本番に自動反映        本番に自動反映
```

**git push するだけで、フロントエンドとバックエンド両方が自動デプロイされます。**

### Q: 問題があった場合、元に戻せますか？

**A:** はい。Railway/Vercel ともに過去のデプロイ履歴を保持しており、ワンクリックで前のバージョンにロールバックできます。

## 8. HPB 固有の仕様

### Q: HPBの規約上、AIで生成した文章を掲載して問題ないですか？

**A:** AI生成テキストについて、HPBは明確な禁止規定を設けていません（2026年3月時点）。ただし、本ツールは「AIが下書き → 人が確認・編集 → 掲載」の設計思想であり、AI出力をそのまま掲載することは推奨していません。

**注意: AI生成の画像はHPB掲載禁止です**（2024年2月リクルート公式発表）。本サービスはテキスト生成に特化しています。

### Q: HPBの文字数制限にはどう対応していますか？

**A:** HPB独自の全角文字数カウント（半角は0.5文字としてカウント）に対応した専用カウンターをフロントエンドに内蔵しています。生成時にもプロンプトで文字数制限を指示し、生成後のカウンターでリアルタイム確認できます。

### Q: HPBへの反映方法は？

**A:** コピー&ペースト方式です。HPBは公開APIを提供していないため、自動連携はできません。生成テキストの「コピー」ボタンをワンクリック → HPBサロンボードに貼り付け、という流れです。
